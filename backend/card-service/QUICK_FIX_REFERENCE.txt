# Quick Fix Reference - UUID Type Conversion

## Error Fixed
```
java: incompatible types: java.lang.String cannot be converted to java.util.UUID
```

---

## What Was Changed

### 1. MerchantVaultAccountService.java
**Line 37**: Changed return type to UUID
```java
- public String getVaultAccountForMerchant(UUID merchantId)
+ public UUID getVaultAccountForMerchant(UUID merchantId)
```

### 2. CreditCardPaymentFlow.java  
**Line 61**: Changed variable type to UUID
```java
- String merchantVaultAccount = 
+ UUID merchantVaultAccount =
    merchantVaultAccountService.getVaultAccountForMerchant(merchantId);
```

### 3. DebitCardPaymentFlow.java
**Line 60**: Changed variable type to UUID
```java
- String merchantVaultAccount = 
+ UUID merchantVaultAccount =
    merchantVaultAccountService.getVaultAccountForMerchant(merchantId);
```

### 4. TransactionGateway.java (Bonus)
**Added annotation** for clean DI
```java
+ @RequiredArgsConstructor
  public class TransactionGateway { ... }
```

---

## Why It Failed

```
transactionGateway.transfer(
    card.getAccountId(),        // UUID ✅
    merchantVaultAccount,       // String ❌ MISMATCH!
    ...
)
```

The second parameter expects UUID but was receiving String.

---

## Why It Works Now

```
transactionGateway.transfer(
    card.getAccountId(),        // UUID ✅
    merchantVaultAccount,       // UUID ✅ FIXED!
    ...
)
```

Both parameters are now UUID type.
